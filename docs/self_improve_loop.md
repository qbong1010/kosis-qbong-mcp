# Self-Improve Loop

## 반복 루프
1. 사용자의 분석 요청을 데이터 요구사항으로 쪼갠다.
2. 필요한 출처, 통계표, 행/열 코드, 연도, 지표, 단위를 명시한다.
3. 실제 데이터 확보 여부를 조회한다.
4. 확보되지 않은 데이터가 있으면 분석을 중단하고 사용자에게 요청 또는 대안을 제시한다.
5. 확보된 데이터만으로 가능한 분석 범위를 명시한다.
6. 분석 후 통계표 ID, 파라미터, 제외값, 경고, 산출물을 기록한다.
7. 새로 발견한 통계표/코드/실패 사례는 `docs/project_memory.md`와 config에 반영한다.

## KOSIS 사용 체크리스트
- `search_statistics` 또는 통계표 검색으로 후보를 찾는다.
- `get_table_info`/metadata 또는 샘플 호출로 실제 `objL*`, `itmId`, 기간을 확인한다.
- 같은 통계표라도 필요한 분류값을 고정한다. 예: `DT_1K52F01`은 `objL3=0`이 사업체구분 계다.
- `ALL` 호출 결과가 너무 크면 산업코드/지역/사업체구분을 좁혀 조회한다.
- 표 이름이 비슷해도 조사명과 모집단을 비교한다.

## 분석 중단 조건
- 비교 대상 중 한쪽에 기준연도 데이터가 없다.
- 성장률 시작연도 또는 종료연도 중 한쪽 데이터가 없다.
- 전국과 지역의 산업분류 수준이 다르다.
- 사업체수와 종사자수가 섞였다.
- 단위가 다르다.
- 비공표값을 0으로 대체해야만 계산 가능한 상황이다.
