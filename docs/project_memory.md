# Project Memory

## 핵심 결정
- 강릉시 지자체 사업체조사 엑셀은 원본 파일로 보존하고, 분석용으로 DuckDB/Parquet long-form 테이블을 생성한다.
- 로컬 원자료 저장 위치: `data/sources/local_gov/gangneung/business_survey/2024/raw/`.
- 분석 저장소: `data/processed/local_business_survey.duckdb`.
- 강릉 로컬 중분류 총괄 canonical source는 `3 산업중분류, 종사자규모 및 읍면동별 사업체수, 종사자수.xlsx`.
- 전국 LQ 기준자료는 KOSIS `DT_1K52F01`을 사용한다.

## 시행착오
- 처음에는 `DT_1FS1101`을 사용했다. 이 표는 강릉시까지 내려가지만 `광업제조업조사`, `10명 이상` 기준이라 사업체조사 전체 기준과 맞지 않는다.
- `DT_1K52F01`은 전국사업체조사 계열이고 지자체 사업체조사 원자료와 모집단이 더 잘 맞는다.
- `DT_1K52F01`은 강릉시 시군구 값을 제공하지 않아, 강릉 값은 지자체 수급 원자료를 써야 한다.
- KOSIS API는 `format=json`이어도 JavaScript object literal 비슷한 응답을 줄 수 있어 별도 파싱이 필요했다.
- 같은 KOSIS 통계표라도 `objL1`, `objL2`, `objL3`, `itmId`를 잘못 세팅하면 전혀 다른 모집단/분모가 된다.

## 숫자로 확인한 차이
- 2024 전국 제조업 전체 `DT_1K52F01`: 사업체 509,467개, 종사자 4,159,656명.
- 2024 전국 제조업 전체 `DT_1FS1101`: 사업체 73,570개, 종사자 3,037,018명.
- 강릉 로컬 원자료 제조업 전체: 사업체 1,492개, 종사자 7,208명.
- 기존 `DT_1FS1101` 강릉 제조업: 사업체 119개, 종사자 3,784명.

## 원칙
- 분석 전에는 반드시 데이터 확보 가능성, 연도 일치, 지표 일치, 단위 일치, 산업분류 수준 일치를 확인한다.
- 필요한 데이터가 없으면 계산하지 않는다.
- 누락 데이터와 가능한 대안을 사용자에게 명시한다.
